# the-ai-theorem — Canon (v0.0.3)

> **Canon is the standard method for human–AI collaboration** — so that any role-cleared
> participant, whatever their starting capability, can produce work at the project's quality
> standard by doing the work the way Canon defines. Domain-neutral and vendor-neutral.

## The idea in one line

AI advances faster than people can keep up with. Canon absorbs the advance **once, as a
shared method**: the AI **leads** the work by Canon, or **accepts the human's lead and
evolves** Canon into a project-optimized form — and teaches and tools the human along the
way. A participant reaches the standard through the **method**, not through prior expertise.

## What Canon is (and is not)

Canon defines **what good human–AI collaboration requires and how that is checked**. It is
**not** the software that runs it. The runnable mechanism (executable procedures, checkers,
runtime state, plug-in resolution) is a separate **Tool** that *conforms to* Canon.

## Status

Draft. Canon was hard-reset in v0.0.3 from a bundled lean-core (doctrine + runnable
mechanism) down to the **pure standard method**. The runtime mechanism is **not** part of
Canon — it is built as a separate **Tool** in its own repository. Ecosystem governance
operations (neutral body, certification, charter) are intentionally **out of Canon** —
handled separately.
